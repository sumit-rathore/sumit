RexCtrl

Offset: 0x20000328

REX control register. Note: fields in this register apply only to REX configurations.

Bits Field Name Description R/W Reset Value Value Type
0 SofEn SOF synchronization and generation enable field.
0 = do not generate SOFs.
1 = generate SOFs based on the setting in the SOFAvgMode field		 read-write 0x0 binary
1 SchEn REX scheduler enable field.
0 = REX scheduler is disabled.
1 = REX will schedule packets in the Scheduled queue to either the uFrame or ISOC frame queues.		 read-write 0x0 binary
2 TxEn REX transmit enable field.
0 = REX will not transmit anything
1 = REX will transmit packets in the SOF, Async, uFrame, and ISOC frame queues.		 read-write 0x0 binary
3 SOFAvgMode 0 = Average mode. SOF is synchronized to the received SOF
1 = Fixed mode. SOF is generated base on the SOFMinTime rate and is not synchronized to the received SOF
In Average mode Rex waits for the first downstream before it starts generating the SOF's to the device. In Fixed mode Rex will generate its own SOF once this field, TxEn, and SofEn are set.		 read-write 0x0 binary
7:4 DbgTxState This is the current state of the xrex tx ctrl which deals with the sending and receiving of packets to/from the device.
State
0 : Check Cache : Wait for a packet to be written to the cache and then send to the device
1 : Wait for read header : Wait for the frame header from the cache to be read
2 : Wait Start Tx : If we are going to send an Out or Setup then wait for the data packet to be written to the data queue before sending the packet
3 : Get Retry Q : Get a Queue to save a data packet to for retrying
4 : Start Tx : Send the packet for the frame header read from the cache to the device
5 : Retry : Retry the last packet sent
6 : Wait for tx complete : Wait until the packet has been sent to the device
7 : Wait for response : Wait for an errored response or the pid of the response from the device
8 : Wait for end of response : Wait for the end of the response
9 : Wait for send ack to device complete : Wait for the ack to be sent to the device
10 : Wait for Sof : (default) Wait for Sof from Xcrm
11 : Tx Sof : Wait until SOF has been sent
12 : Dump Pkt : For flow control dump the In packets to be sent to the device
13 : Wait for timeout : Wait for a timeout from the device
14 : Send Upstream : send an upstream packet
15 : Wait Xurm Done : Wait for the xurm to finish receiving the packet		 read-only 0xA unsigned
10:8 SofAvgCnt This sets when the SOF should be adjusted based on the frame count. A value of 1 means that every frame the SOF is adjusted. A value of 2 means the SOF is adjusted every second frame. read-write 0x1 unsigned
24 Msa_Retry_Disable Setting this bit will disable the retrying of MSA packets in hardware and will send all responses back to the cpu read-write 0x0 unsigned
